# Bulk / Marketing Email Fingerprints

Bulk/marketing email has a fingerprint across headers, envelope, body, and infrastructure. Here's the cheat sheet.

## Headers (most reliable)

- **`List-Unsubscribe`** — RFC 2369 header pointing to a `mailto:` or HTTPS unsubscribe. Required by Gmail/Yahoo for senders >5K/day. Personal mail never has this.
- **`List-Unsubscribe-Post: List-Unsubscribe=One-Click`** — RFC 8058 one-click unsubscribe. Pure marketing signal.
- **`List-Id`** — names the mailing list.
- **`Precedence: bulk`** (or `list`, `junk`) — explicit "this is bulk" declaration.
- **`Feedback-ID`** — Gmail/Google Postmaster format `campaign:template:sender:platform`. Pure ESP signal.
- **`X-Mailer`** / **`X-Mailgun-*`** / **`X-SG-EID`** (SendGrid) / **`X-Mandrill-*`** / **`X-CSA-Complaints`** / **`X-Campaign`** / **`X-Postal-MsgID`** — vendor fingerprints.
- **`Auto-Submitted: auto-generated`** — generated by software.

## Envelope / routing

- **VERP / per-recipient bounce addresses**: `Return-Path` like `bounces+abc123@...` or `dwknqk@psrp.domain.com` (random local part on a `bounce.` / `psrp.` / `mta.` subdomain). Personal mail uses the sender's normal address.
- **Different `From:` domain than `Return-Path:` / DKIM `d=`** — ESP sends on behalf of brand.
- **DKIM signed by an ESP domain** (`d=mailgun.org`, `d=sendgrid.net`, `d=amazonses.com`, `d=mcsv.net` Mailchimp).
- **Sending IPs in known ESP ranges** (Mailgun 69.x, SendGrid 167.x, SES, Mailchimp).
- **`Received: from api`** or `from web` — message injected via HTTP API, not SMTP from a mail client.

## Body

- **`<img width="1" height="1">`** open-tracking pixel.
- **All links wrapped through a click-tracking domain** (`click.brand.com`, `r.brand.com`, `email.brand.com/ls/click?...`, `*.list-manage.com`, `awstrack.me`, `sendgrid.net/ls/click`).
- **Pre-header text** in hidden `<div style="display:none">` (the gray text Gmail shows next to the subject).
- **MSO conditional comments** (`<!--[if mso]>`) for Outlook compatibility — only template builders use these.
- **`<table>`-based layout** (still the standard for HTML email; personal mail is `<div>`/text).
- **Padding characters in subject/preheader**: invisible Unicode `‌` `͏` `‍` to pad inbox preview.
- **Footer with physical address** (CAN-SPAM requirement) + unsubscribe link.
- **`{{merge}}` artifacts** when templating fails — `Hi {{first_name}},`.

## Volume / pattern signals (need history, not one message)

- Same template hash sent to many recipients.
- Sent at a round time (cron-scheduled: 09:00:00 UTC exactly).
- "Batch" / "weekly digest" / "[NEWSLETTER]" in subject.
- Sender sends only outbound, never receives replies on that address (`noreply@`, `no-reply@`, `notifications@`).

## Cold outreach (a sub-category)

Cold B2B outreach often *intentionally hides* bulk markers to look personal:

- No `List-Unsubscribe`, no tracking pixel, plain text only.
- Sent through Postal / Smartlead / Instantly / Lemlist via API.
- Random envelope local part (the `dwknqk@psrp...` pattern).
- Subject like `Re: ...` or `Quick question` to fake a thread.
- "Halfway through the [month] batch" / "Circling back" / "Following up on my last email."
